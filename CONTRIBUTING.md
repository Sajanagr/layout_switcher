# Contributing to KBFix

Спасибо, что хотите помочь проекту ❤️

Ниже — правила, чтобы изменения были понятными и легко принимались.

---

## 1) Как предложить улучшение

1. Проверь Issues (возможно, тема уже обсуждается).
2. Создай новый Issue:
   - что не так / что нужно
   - как воспроизвести (если баг)
   - ожидаемое поведение
   - фактическое поведение
   - версия ОС (Ubuntu/Debian), GNOME/Wayland или X11

---

## 2) Как сделать Pull Request

### Ветка
- Создавай отдельную ветку от `main`
- Название ветки:
  - `feature/<short-name>`
  - `fix/<short-name>`
  - `docs/<short-name>`

### Коммиты
- Сообщения в стиле:
  - `feat: ...`
  - `fix: ...`
  - `docs: ...`
  - `chore: ...`

### Требования к коду
- Код должен быть читабельным.
- Комментарии — на русском языке (как в проекте сейчас).
- Не добавлять лишние зависимости без необходимости.

---

## 3) Локальный запуск (dev)

### Установка зависимостей ОС (Ubuntu/Debian)
```bash
sudo apt update
sudo apt install -y python3 python3-gi gir1.2-ayatanaappindicator3-0.1 python3-pynput python3-pyperclip
````

### Запуск из репозитория

```bash
python3 -m kbfix
```

---

## 4) Структура проекта (кратко)

* `kbfix/` — пакет приложения
* `assets/` — иконки
* `desktop/` — desktop-файлы
* `debian/` — упаковка .deb
* `experiments/` — эксперименты (не часть релиза)

---

## 5) Что особенно приветствуется

* улучшение поведения под Wayland
* подключение словарей для автокоррекции
* настройки горячей клавиши
* уведомления / логи и просмотр логов из tray
* упаковка и релизы
